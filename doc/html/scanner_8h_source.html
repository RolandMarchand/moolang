21 /*
22  * The scan struct is the final output of the scanner. It contains the
23  * source code of the program and a dynamic array of tokens.
24  *
25  * To get the scan struct, call the `scan_init` function.
26  * To delete the scan, call the `scan_del` function. Not doing so will
27  * result in a memory leak.
28  *
29  * The source is a structure composed of a `string` character array, a
30  * `size` integer measuring physical size, and a `file_descriptor`. The source is managed
31  * automatically.
32  *
33  * The dynamic array of tokens is similar to the source, composed of an
34  * `array` of tokens, a `size` integer measuring the physical size of
35  * the array, and a `count` integer, counting the number of tokens. The
36  * array of tokens is managed automatically.
37  *
38  * Tokens of the dynamic array are structures being made of their
39  * `type` (TypeToken), the `line` on which they appear in the source
40  * code, and a `lexeme`. The `lexeme` is a custom structure called a
41  * substring.
42  *
43  * Substrings are composed of two char pointers `start` and `end`
44  * pointing to a section of a string. You can use `sbstrcpy` to
45  * write the contents of a substring in a user-defined character array
46  * of size `SUBSTRING_LENGTH(sbstr) + 1`. There is a
47  * `PRINT_SUBSTRING(sbstr)` macro, to print the substring to STDOUT.
48  */
